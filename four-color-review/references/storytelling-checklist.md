# Storytelling Review Checklist

## Closure checklist

A story is closed only when all of these are visible:
- a clear initiator
- a clear trigger or context
- one or more key actions
- a business response
- a result that can be observed or consumed
- a closure point for failure, timeout, or manual intervention if relevant

## Blocking conditions

Stop and ask when any of these are missing:
- no accountable role
- no result owner
- no terminal state
- no handler for exception or timeout
- no object carrying a critical rule
